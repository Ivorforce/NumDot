# === Section: NumDot precision edges (file issue) ===
# NumDot's va::arange computes num after converting all inputs to its
# process_dtype (the common type). When start/stop are huge ints (>2^53)
# and step is a float, both ints round independently to float64 BEFORE
# the subtraction, losing 8+ bits of precision in the diff. numpy avoids
# this by doing the int subtraction first. Hypothesis finds inputs that
# exceed array-api-tests's ±10% size tolerance.
# Pure-int and pure-float arange now work correctly (200+ examples) thanks
# to the integer-arithmetic and direction-mismatch fixes.
array_api_tests/test_creation_functions.py::test_arange
